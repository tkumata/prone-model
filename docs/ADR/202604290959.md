# ADR: エージェント停止時 ESP-IDF ビルドハーネスを共有化する

## Status

Accepted

## Context

Codex CLI の `Stop` と Copilot CLI の `agentStop` の両方で、エージェント作業停止時に ESP-IDF build を実行したい。

hook ごとに `source "$IDF_PATH/export.sh" && idf.py build` 相当の処理を重複実装すると、ESP-IDF パス、ログ保存、エラー要約、終了コードの挙動が分岐しやすい。

## Decision

`.agent-hooks/check_build.sh` を共有ハーネスとし、`.codex/hooks.json` と `.github/hooks/hooks.json` は共有ハーネスを呼び出すだけにする。

ESP-IDF 環境は `IDF_PATH` を優先し、未設定時は `$HOME/.espressif/v6.0/esp-idf` を候補にする。

`IDF_TOOLCHAIN_PATH` には依存しない。

既存 `build/CMakeCache.txt` に `PYTHON` が記録されている場合は、その Python 環境を `IDF_PYTHON_ENV_PATH` として優先する。

## Consequences

- CLI ごとの差分は hook 設定に閉じる
- build 実行とログ処理は 1 箇所で保守できる
- ESP-IDF v6.0 前提を明示できる
- hook 設定だけを見ても詳細挙動は分からないため、共有ハーネスの仕様文書を同期して保守する必要がある
