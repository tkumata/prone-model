# 受け入れ基準

## 1. 目的

本基準は、実装着手前にドキュメントが十分に閉じているか、また収集仕様として不足がないかを判断するための基準である。

## 2. ドキュメント完備条件

以下が存在し、相互に矛盾しないこと。

- 要件定義
- 設計
- 仕様
- 収集運用規程
- ラベル付け基準
- データ品質基準
- 命名規則

## 3. 必須合意事項

以下が文書化されていること。

- `subject_id` 必須
- `session_id` 必須
- 画像本体保存必須
- 学習利用可否の管理方法
- 除外理由の語彙
- 被写体単位分割方針
- 収集条件の最低目標
- ラベル付け基準
- PC 上でのモデル生成前処理条件
- `ESP-DL` 用成果物の定義
- 実機受け入れ試験条件
- 実機合格基準

## 4. 実装着手許可条件

以下をすべて満たした場合のみ実装に進んでよい。

- 保存対象列が固定されている
- UI 必須入力が固定されている
- API 入出力が固定されている
- 学習対象外サンプルの扱いが固定されている
- 命名規則が固定されている
- 運用ルールが固定されている
- 分割エクスポート仕様が固定されている
- PC 側の `train / val / test` 分割条件が固定されている
- `float` モデルから `ONNX`、`ESP-DL` 形式までの生成物が定義されている
- 判定閾値の決定方法と固定条件が定義されている
- 実機確認用集合の分離条件が定義されている
- 実機合格基準が数値で定義されている

## 5. 不許可条件

以下が 1 つでもある場合、実装着手を禁止する。

- `subject_id` 要件が曖昧
- 曖昧姿勢の扱いが曖昧
- `non_prone` の定義が曖昧
- 学習利用可否のルールが未定
- 除外理由語彙が未定
- 被写体単位分割ルールが未定
- 保存列が未確定
- PC 側入力サイズが未確定
- モデル出力順が未確定
- 閾値決定方法が未定
- 実機確認用集合の扱いが未定
- 実機合格基準が未定

## 6. データセット受け入れ条件

収集済みデータセットは、以下を満たした場合に受け入れる。

- 必須メタデータ欠損がない
- `subject_id` 単位分割可能
- 曖昧サンプル除外可能
- 画像監査可能
- 画像と CSV 対応が正しい
- 分割エクスポートで全件取得可能
- `Freenove ESP32-S3 WROOM CAM` 由来データのみで PC モデル生成に進める
- 同一条件で `ESP-DL` 用モデルを再生成できる
- 実機確認用集合で最終モデルが合格基準を満たす

## 7. 完了判定

本プロジェクトのドキュメントフェーズは、上記すべてを満たし、追加の未確定項目が残っていないときに完了とする。
