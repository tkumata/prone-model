# 要件定義

## 1. 目的

本プロジェクトの目的は、乳幼児のうつ伏せ状態を検出可能な AI モデルを作成し、最終的に `ESP-DL` で扱える `.espdl` 形式へ変換することである。
入力情報は顔検出から得られる顔情報のみに限定し、画像そのものをモデル入力として使わない。

## 2. 背景

乳幼児のうつ伏せ判定を、マイコンで扱える軽量なモデルとして実現したい。
そのため、`Freenove ESP32-S3 WROOM CAM` と `ESP-DL` を用いて数値データを収集し、PC 側で学習し、最終的に `.espdl` 形式へ落とし込む。

## 3. ゴール

- 顔特徴量のみで `うつ伏せ` と `非うつ伏せ` を判定できる学習済みモデルを作成する
- 学習済みモデルを `.espdl` 形式へ変換できる
- マイコンで学習用データを収集できる
- 収集から前処理、学習、評価、変換までの手順を再現可能にする

## 4. 非ゴール

- 画像を保存して画像分類モデルを作ること
- 乳幼児以外を対象とした汎用姿勢検出
- 全身姿勢や骨格推定を用いた判定
- 即時アラート機能の完成

## 5. 前提

- ハードウェアは `Freenove ESP32-S3 WROOM CAM` を利用する
- 顔関連モデルは `ESP-DL` を利用する
- 取得可能な顔特徴点は `左目` `右目` `鼻` `左口角` `右口角` の5点である
- カメラ位置は乳幼児に対して斜め上で固定する
- うつ伏せ状態は顔の一部が見えている前提で扱う
- データ収集時のラベル付けは人間が映像を確認して行う
- 画像は保存せず、数値データとメタ情報のみを保存する
- Wi-Fi は既存 AP に接続する `STA` モードとする
- `SSID` とパスフレーズは `sdkconfig` に保持し、リポジトリには含めない

## 6. 用語

- 顔検出: 顔の存在と顔枠を取得する処理
- 顔特徴点: `左目` `右目` `鼻` `左口角` `右口角` の座標
- 顔未検出: 顔枠が取れない状態
- 収集セッション: 同一人物、同一環境、同一時間帯で連続して収集した単位
- 有効サンプル: `face_detected = 1` で、必要なメタ情報を持つサンプル

## 7. 判定方針

- 姿勢分類モデルの学習対象は `face_detected = 1` のサンプルのみとする
- `face_detected = 0` はログとして保存するが、姿勢分類モデルには直接学習させない
- 推論時は `顔検出` と `姿勢分類` を分離する
- 顔未検出はモデル出力ではなく、運用ルールで補助的に扱う

## 8. デフォルト動作

- Web 画面には映像プレビュー、現在の検出状態、ラベル付き保存ボタン、CSV エクスポート、CSV リセットを表示する
- 保存は連写ではなく、ボタン押下時点の1サンプルのみを保存する
- ラベルは `うつ伏せ = 1` `非うつ伏せ = 0` の2値とする
- 顔が複数検出された場合は保存しない
- 顔未検出の場合もログとして保存する
- 顔特徴点欠損は座標 `0` ではなく、番兵値と可視フラグで表す

## 9. 数値要件

### 9.1 収集時の初期採用値

- ストリーム解像度は初期値として `320 x 240` を採用する
- 保存対象は最大1顔とする
- 顔検出信頼度の初期保存閾値は `0.40` とする

### 9.2 データ量の初期目標

- 成立性確認用として各クラス `1000` サンプル以上を集める
- 初期モデル作成用として各クラス `3000` サンプル以上を集める
- 収集セッションは最低 `10` セッション以上とする
- 照明条件は最低 `3` 条件以上を含める

### 9.3 モデル評価の初期目標

- `うつ伏せ` の再現率は `0.90` 以上を初期目標とする
- `うつ伏せ` の適合率は `0.80` 以上を初期目標とする
- 評価分割は行単位ではなくセッション単位または被験者単位で行う

## 10. 入力と出力

### 10.1 学習入力

- 顔枠基準で正規化した5点座標
- 各点の可視フラグ
- 顔枠情報
- 顔検出信頼度

### 10.2 学習出力

- `1 = うつ伏せ`
- `0 = 非うつ伏せ`

### 10.3 運用時の補助状態

- `face_detected = 0`

## 11. 保存すべき情報

- `timestamp`
- `session_id`
- `label`
- `face_detected`
- `face_score`
- `bbox_x`
- `bbox_y`
- `bbox_w`
- `bbox_h`
- 5点座標
- 5点可視フラグ

## 12. 実装対象

### 12.1 マイコン側

- カメラ初期化
- Wi-Fi `STA` 接続
- 顔検出と顔特徴点抽出
- Web サーバ
- 映像ストリーミング
- ラベル付き保存 API
- CSV エクスポート API
- CSV リセット API
- SD カード書き込み

### 12.2 PC 側

- CSV 検証スクリプト
- 前処理スクリプト
- 学習スクリプト
- 評価スクリプト
- `.espdl` 変換スクリプト
- 推論確認用の検証スクリプト

## 13. エラー時の挙動

- Wi-Fi 接続失敗時は自動再試行する
- SD カード書き込み失敗時は保存を失敗として扱い、成功扱いにしない
- 顔が複数検出された場合は保存を拒否し、理由を表示する
- CSV リセットは明示的な確認操作を要求する
- 顔枠が無効な場合は正規化を行わず、保存行を `face_detected = 0` として記録する

## 14. 成功条件

- 学習用データ収集機能が安定して動作する
- データに必要な列が欠落なく保存される
- 顔特徴量のみで初期目標を満たすうつ伏せ判定モデルが得られる
- 学習済みモデルを `.espdl` へ変換し、`ESP-DL` で利用可能である
