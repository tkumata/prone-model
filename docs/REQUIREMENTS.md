# 要件定義

## 1. 目的

`Freenove ESP32-S3 WROOM CAM` を用いて、乳幼児の `うつ伏せ検知モデル` を学習するために十分な品質を持つ画像データセットを収集する。

本プロジェクトの最優先要件は「画像を保存できること」ではなく、「後段のモデル学習、評価、再現実験に耐えるデータセットを確実に作れること」とする。

## 2. ゴール

- 既存 Wi-Fi AP に `STA` 接続し、ESP32 上で Web サーバを起動できる
- ブラウザからライブストリームを見ながらラベル付き撮影できる
- JPEG 画像と学習必須メタデータを SD カードへ保存できる
- 保存済みデータ一式を PC 側へエクスポートできる
- 学習データとして使えないサンプルを識別または除外できる
- 被写体リークを防いだ学習/検証/評価分割が可能なメタデータを必ず残す
- `Freenove ESP32-S3 WROOM CAM` で収集したデータだけを使い、PC 上で `ESP-DL` 用モデル生成に進める
- PC 側で前処理、学習、量子化、`ESP-DL` 形式変換に必要な情報を欠損なく取り出せる
- PC 上で生成した `.espdl` を最終的にマイコンへ載せ、乳幼児の `prone / non_prone` を安定して判定できる

## 3. 非ゴール

- ESP32 上で姿勢推論を実行すること
- 自動ラベル付けを行うこと
- モデル学習自体を ESP32 上で完結させること
- PC 学習基盤の実装方法や学習コードの詳細をこの段階で確定すること
- 本番運用前に曖昧な閾値や未固定前処理のまま実装へ進むこと

## 4. 絶対遵守要件

以下は必須であり、省略不可とする。

- `subject_id` を全サンプルで必須とする
- `session_id` を全サンプルで必須とする
- `location_id` を全セッションで必須とする
- `lighting_id` を全セッションで必須とする
- `camera_position_id` を全セッションで必須とする
- `annotator_id` を全セッションで必須とする
- `timestamp_ms` を全サンプルで必須とする
- `board_name` を全サンプルで必須とし、値は `Freenove ESP32-S3 WROOM CAM` で固定する
- `label` とは別に、学習利用可否を表す列を必須とする
- 曖昧サンプルや失敗サンプルを負例に混入させない
- 学習/検証/評価の分割は `subject_id` 単位で行えるようにする
- 後からラベル監査できるよう、画像本体を必ず保存する
- PC 側で `ESP-DL` モデル生成に使う前処理条件をドキュメント上で固定する
- PC 側で `float` 学習済みモデルと `ESP-DL` 変換済みモデルを同じデータ分割から再生成できること
- PC 側前処理とマイコン側前処理は画素並び、色順、リサイズ条件を一致させること
- `.espdl` 化後の判定結果が PC 側量子化参照結果から実用上乖離しないこと
- 実機受け入れ試験で `Freenove ESP32-S3 WROOM CAM` のライブ入力から `prone / non_prone` を判定できること

## 5. ラベル要件

### 5.1 姿勢ラベル

- `1 = prone`
- `0 = non_prone`

### 5.2 学習利用可否

姿勢ラベルとは別に、少なくとも以下を持つこと。

- `is_usable_for_training`
  - `1`: 学習利用可
  - `0`: 学習利用不可
- `exclude_reason`
  - 利用不可理由を保存する

### 5.3 除外理由の最低分類

以下を最低限扱えること。

- `ambiguous_pose`
- `face_not_visible`
- `subject_missing`
- `multiple_subjects`
- `motion_blur`
- `poor_lighting`
- `annotation_error`
- `other`

## 6. 必須メタデータ要件

各画像に対して、最低でも以下を保存する。

- `capture_id`
- `timestamp_ms`
- `subject_id`
- `session_id`
- `label`
- `label_name`
- `is_usable_for_training`
- `exclude_reason`
- `location_id`
- `lighting_id`
- `camera_position_id`
- `annotator_id`
- `notes`
- `image_path`
- `image_bytes`
- `frame_width`
- `frame_height`
- `pixel_format`
- `jpeg_quality`
- `board_name`

## 7. 収集条件要件

学習データとして成立させるため、収集運用で以下を守る。

- 複数被写体を含める
- 各被写体について `prone` と `non_prone` の両方をできる限り含める
- 照明条件を複数含める
- 撮影距離と画角の偏りを避ける
- 同一セッションの連写だけで全体件数を満たさない
- 学習利用可サンプルでは被写体の体幹が確認できること
- 学習利用可サンプルでは被写体の短辺方向サイズが入力画像短辺の 30% 以上を目安とする
- 学習利用可サンプルでは頭部から体幹までの主要部位が画面内に収まること
- 強い逆光、白飛び、黒つぶれで姿勢判定に必要な輪郭が失われた画像は学習利用可にしない

## 8. 分割要件

- 学習/検証/評価分割は `subject_id` 単位で行う
- 行単位ランダム分割は禁止する
- `session_id` は補助集計単位として使う
- `subject_id` が欠損したサンプルは学習データセットに採用しない

## 9. UI 要件

Web 画面 `/` には以下を必須配置する。

- カメラストリーミング
- 撮影ボタン
- `うつ伏せ / 非うつ伏せ` ラジオボタン
- エクスポートボタン
- SD カードリセットボタン

学習データ品質のため、追加入力として以下を必須とする。

- `subject_id`
- `session_id`
- `location_id`
- `lighting_id`
- `camera_position_id`
- `annotator_id`
- `学習利用可否`
- `除外理由`
- `notes`

## 10. API 要件

- `GET /`
  Web UI を返す
- `GET /stream`
  カメラストリームを返す
- `GET /api/status`
  現在状態と収集件数を返す
- `POST /api/capture`
  画像と必須メタデータを保存する
- `GET /api/export/manifest`
  エクスポート対象一覧とページ情報を返す
- `GET /api/export/metadata`
  `metadata.csv` を返す
- `GET /api/export/image`
  指定画像を 1 件返す
- `POST /api/reset`
  データセット全体を削除する

## 11. エクスポート要件

- 単一巨大レスポンスで全件返さないこと
- `metadata.csv` と画像取得 API を分離すること
- 画像は 1 件ずつ再試行可能に取得できること
- 通信断後に途中再開できること
- エクスポート後に画像と CSV の対応が失われないこと
- PC 側でそのまま監査、分割、学習前処理に入れること
- PC 側で `train / val / test` 分割用一覧を機械生成できるだけの列を含むこと
- PC 側で `ONNX` 変換入力として使う学習済みモデルの再現条件を追跡できること

## 12. PC での `ESP-DL` モデル生成要件

- モデル生成は PC 上で行い、ESP32 側は収集とエクスポートだけを担う
- 学習元画像は `Freenove ESP32-S3 WROOM CAM` で取得した JPEG のみを使う
- 学習入力は RGB 3 チャンネルに変換し、初期仕様では `96 x 96` に正規化する
- 初期の分割比率は `train=0.70`, `val=0.15`, `test=0.15` とする
- 分割再現用の乱数シード既定値は `42` とする
- 初期学習条件は `epoch=20`, `batch_size=32`, `learning_rate=0.001` とする
- 学習対象は `is_usable_for_training=1` かつ `exclude_reason=""` の行だけとする
- 学習/検証/評価分割は `subject_id` 単位で固定し、同一 `subject_id` を複数分割へ跨がせない
- PC 側では少なくとも `学習用一覧`, `評価結果`, `float` 学習済みモデル, `ONNX`, `ESP-DL` 変換済みモデルを成果物として残せること
- 2 クラス分類として扱い、出力順は `0=non_prone`, `1=prone` で固定する
- 量子化後モデルでも、どの分割と前処理条件で生成したかを追跡できること
- `float` モデル、量子化モデル、`.espdl` 実機結果を同一評価集合で比較できること
- 検証データで決めた判定閾値を評価データと実機受け入れ試験で固定利用すること
- 初期の判定閾値既定値は `0.50` とし、検証結果で更新した後は固定すること
- `.espdl` 生成後にクラス順、入力サイズ、色順が変わっていないことを確認できること

## 13. 実機検知要件

- 推論対象ハードは `Freenove ESP32-S3 WROOM CAM` とする
- 実機入力は同ボードのライブカメラフレームを使う
- 実機側の前処理は PC 側学習前処理と同一条件であること
- 実機出力は `non_prone`, `prone` の 2 クラススコアまたは確率相当値を返せること
- 判定閾値は検証データで決め、実機評価前に固定すること
- 実機推論結果は PC 側量子化参照結果と高一致であること
- 実機では少なくとも `推論時刻`, `推論クラス`, `スコア`, `入力画像識別子または時刻` を検証記録として残せること

## 14. エラー時要件

- カメラ異常時は撮影を拒否する
- SD カード異常時は保存、エクスポート、リセットを拒否する
- 必須メタデータ欠損時は保存を拒否する
- 不正な `label` や `exclude_reason` は保存を拒否する
- 不正な `location_id`, `lighting_id`, `camera_position_id`, `annotator_id` は保存を拒否する
- PC 側で依存ライブラリ未導入時は不足パッケージ名を列挙して停止する
- PC 側で学習対象画像が 1 枚もない場合は学習を開始しない
- PC 側で `subject_id` が 3 件未満の場合は `train / val / test` 分割を拒否する
- PC 側で `val` または `test` に対象 `subject_id` を割り当てられない場合は分割比率を変えずに停止する
- PC 側で前処理条件、クラス順、閾値のいずれかが未固定なら `.espdl` 生成工程へ進めない
- `.espdl` 化後に PC 量子化参照との差分が許容範囲を超えた場合は実機導入を禁止する

## 15. 成功条件

以下をすべて満たしたときのみ、本プロジェクトの収集仕様は成立とみなす。

- 全サンプルに `subject_id` と `session_id` が存在する
- 全サンプルに `location_id`, `lighting_id`, `camera_position_id`, `annotator_id` が存在する
- 曖昧サンプルが `non_prone` 正例として混入しない
- 学習利用可否と除外理由でフィルタリング可能である
- 分割エクスポートしたデータだけで被写体単位分割ができる
- 保存済み画像を人間が監査できる
- PC 上で `ONNX` と `ESP-DL` 用モデル生成に必要な前処理条件と出力定義が確定している
- `float` モデル、量子化参照、`.espdl` 実機結果の比較で品質劣化が許容範囲内である
- 最終的に `Freenove ESP32-S3 WROOM CAM` 実機上で `prone / non_prone` を安定検知できる
