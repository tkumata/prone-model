# 実行計画

## 1. 完了条件

- マイコン側でラベル付き数値データを安定保存できる
- 学習データを検証、前処理、分割できる
- 初期モデルを学習して評価できる
- 採用モデルを `.espdl` 化できる

## 2. 事前準備

### 2.1 開発環境

- `VS Code`
- `ESP-IDF` 拡張
- `ESP-IDF`
- `ESP-DL`
- PC 側の学習環境
- SD カードの初期化

### 2.2 収集準備

- カメラ設置位置の固定
- 収集セッション命名規則の決定
- ラベル付け手順の確定
- 保存可否の判断基準の確定

## 3. フェーズ1: マイコンの収集基盤を作る

### 3.1 実装するもの

- カメラ初期化処理
- Wi-Fi `STA` 接続処理
- 顔検出処理
- 特徴点抽出処理
- HTTP サーバ
- Web 画面
- CSV 追記処理
- CSV エクスポート処理
- CSV リセット処理

### 3.2 確認すること

- 顔枠と5点が取得できるか
- `face_score` が取得できるか
- 1回の保存操作で1行だけ記録されるか
- 複数顔や保存失敗が適切に弾かれるか

### 3.3 成果物

- 収集ファームウェア
- `prone_samples.csv`

## 4. フェーズ2: データ収集ルールを固める

### 4.1 決めること

- `session_id` の命名規則
- 1セッションあたりの目標サンプル数
- 非うつ伏せの含め方
- 暗所、部分遮蔽、横向きの扱い

### 4.2 実施すること

- 成立性確認用に各クラス `1000` サンプル以上を収集する
- セッションを `10` 以上確保する
- 照明条件を `3` 条件以上含める

### 4.3 注意点

- 同一姿勢の偏りを避ける
- 収集中の画角ずれを記録する
- 曖昧サンプルは無理に保存しない

## 5. フェーズ3: データ検証と前処理を作る

### 5.1 実装するもの

- CSV 検証スクリプト
- 欠損確認処理
- 範囲外値確認処理
- セッション単位の分割処理
- 特徴量生成処理

### 5.2 確認すること

- `face_detected = 1` の有効件数
- クラスごとの件数偏り
- セッションごとの偏り
- 欠損率

### 5.3 成果物

- 学習用データセット
- 検証レポート

## 6. フェーズ4: 初期モデルを学習する

### 6.1 実装するもの

- 学習スクリプト
- 評価スクリプト
- 指標出力処理
- モデル保存処理

### 6.2 実施すること

- `20` 次元入力の初期モデルを学習する
- `train/valid/test` で評価する
- 閾値 `0.50` を基準に混同行列を確認する

### 6.3 判定基準

- `うつ伏せ` 再現率 `0.90` 以上
- `うつ伏せ` 適合率 `0.80` 以上

### 6.4 不達時の対応

- 収集データの偏りを見直す
- `face_score` や `bbox` の寄与を確認する
- 派生特徴量を追加する
- 閾値を再調整する

## 7. フェーズ5: `.espdl` 化する

### 7.1 実装するもの

- モデル変換スクリプト
- 変換後検証スクリプト

### 7.2 実施すること

- 学習済みモデルを固定形式で保存する
- `.espdl` に変換する
- 入出力整合性を確認する

### 7.3 成果物

- `prone_face_classifier.espdl`
- 変換手順書

## 8. フェーズ6: マイコン組み込み前確認

### 8.1 確認すること

- 入力次元が一致しているか
- 前処理式が PC 側と一致しているか
- 出力閾値が固定されているか

### 8.2 判断

- 条件を満たせば採用モデルとする
- 条件を満たさなければフェーズ2またはフェーズ4へ戻る

## 10. リスクと対策

- 画像非保存により誤ラベル監査が難しい
  収集時にセッションメモを残す
- 顔未検出が多すぎると分類モデルの学習が進まない
  画角と照明を見直す
- 収集環境が偏ると汎化しない
  セッションと条件を分散させる
- `.espdl` 化で制約に引っかかる
  初期から軽量モデル構成を維持する
